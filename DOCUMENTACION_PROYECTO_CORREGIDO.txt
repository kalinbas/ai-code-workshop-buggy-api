1. RESUMEN DEL PROYECTO
=======================

Este proyecto es una API HTTP de pedidos creada con Node.js. El repositorio fue
disenado originalmente como una API "rota" para un workshop de debugging asistido
por IA. Su objetivo es practicar lectura de codigo, pruebas, diagnostico de bugs
y correcciones pequenas guiadas por tests.

La API simula una tienda sencilla con:

- catalogo de productos;
- clientes autenticados por token Bearer;
- creacion de ordenes;
- consulta de ordenes;
- calculo de subtotal, impuestos, envio y descuentos;
- reembolsos;
- reporte de ingresos por cliente.

El proyecto ya fue corregido en las areas principales del taller:

- validacion de ordenes;
- reglas de pricing;
- autenticacion y autorizacion.

8. BUGS CORREGIDOS
==================

Se corrigieron 12 comportamientos defectuosos del taller.


8.1 Validacion de ordenes
-------------------------

Archivo principal:

app\server.js

Bugs corregidos:

1. Se aceptaban ordenes vacias.
2. Se aceptaban SKUs desconocidos.
3. Se aceptaban cantidades negativas.
4. Se aceptaban cantidades como string, por ejemplo "2".

Correccion:

Se agrego la funcion validateOrderItems(payload), que valida la orden antes de
calcular precios o guardar datos.

Reglas actuales:

- payload.items debe ser un array;
- payload.items no puede estar vacio;
- cada item debe ser un objeto;
- cada item debe tener un sku string;
- el sku debe existir en el catalogo;
- quantity por defecto es 1 si no se envia;
- quantity debe ser entero;
- quantity debe ser mayor a 0.

Si alguna regla falla, la API responde con error HTTP 422.

Comentarios agregados en el codigo:

- app\server.js, funcion validateOrderItems:
  comentarios "Fix:" explican la validacion de lista vacia, SKU y cantidad.

- app\server.js, ruta POST /orders:
  comentario "Fix:" explica que la validacion se centraliza en la frontera HTTP.


8.2 Pricing, impuestos, cupones y envio
---------------------------------------

Archivo principal:

app\pricing.js

Bugs corregidos:

5. El cupon descontaba subtotal + tax + shipping.
6. El impuesto se calculaba sin considerar el descuento.
7. Los cupones expirados se aceptaban.
8. El envio gratis premium usaba subtotal - discount > 100.

Correccion:

El calculo ahora separa:

- subtotal total;
- subtotal taxable;
- descuento;
- envio;
- impuesto;
- total.

Reglas actuales:

- el descuento de un cupon se aplica solo sobre el subtotal;
- el descuento reduce la base taxable de manera proporcional;
- un cupon desconocido genera error HTTP 422;
- un cupon expirado genera error HTTP 422;
- se respeta minSubtotal del cupon;
- clientes premium tienen envio gratis cuando subtotal >= 100 antes de descuento.

Comentarios agregados en el codigo:

- app\pricing.js:
  comentarios "Fix:" explican el subtotal taxable, validacion de cupones,
  fecha de expiracion, minimo de subtotal, descuento sobre subtotal, envio premium
  y calculo de impuesto despues del descuento.


8.3 Seguridad y autorizacion
----------------------------

Archivos principales:

app\auth.js
app\server.js

Bugs corregidos:

9. Un header malformado como "BearerBearer user-token-1" podia autenticarse.
10. Un cliente podia leer ordenes de otro cliente.
11. Un cliente no admin podia consultar /reports/revenue.
12. Un cliente no admin podia reembolsar ordenes.

Correccion en autenticacion:

En app\auth.js se cambio el parsing del header Authorization.

Antes:

- se usaba replaceAll("Bearer", "").trim();
- esto podia convertir headers malformados en tokens validos.

Ahora:

- Authorization debe ser string;
- debe cumplir el patron estricto Bearer <token>;
- si el formato esta mal, responde 401;
- si el token no existe, responde 403.

Correccion en autorizacion:

En app\server.js:

- GET /orders/:id verifica si el usuario es dueno de la orden o admin;
- POST /orders/:id/refund llama requireAdmin(customer);
- GET /reports/revenue llama requireAdmin(customer).

Comentarios agregados en el codigo:

- app\auth.js:
  comentarios "Fix:" explican validacion de tipo y parsing estricto Bearer.

- app\server.js:
  comentarios "Fix:" explican acceso por dueno/admin, refunds admin-only y reportes
  admin-only.
